# Security Policy

## Supported Versions
We provide security updates for the following versions:

| Version | Supported          |
|---------|--------------------|
| Latest  | ✅                  |
| Older   | ❌                  |

---

## Reporting a Vulnerability

If you discover a security vulnerability in the MJ‑Ahmad Dashboard, please follow these steps:

1. **Do not open a public issue.**  
   Security vulnerabilities should be reported privately to protect users.

2. **Contact us directly:**  
   Send an email to [mjahmad2024@outlook.com](mailto:mjahmad2024@outlook.com) with details of the vulnerability.  
   Include:
   - A clear description of the issue
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

3. **Response timeline:**  
   - We will acknowledge receipt within **48 hours**.  
   - We will provide an initial assessment within **5 business days**.  
   - Fixes or mitigations will be prioritized and released as soon as possible.

---

## Security Best Practices

To help keep the project secure:
- Keep dependencies up to date.
- Avoid committing sensitive information (API keys, passwords).
- Use HTTPS for all external communications.
- Follow secure coding practices (input validation, sanitization, etc.).

---

## Disclosure Policy

We follow a **responsible disclosure** model:
- Vulnerabilities are disclosed publicly only after a fix is available.
- Contributors who report issues responsibly will be credited in release notes.

We appreciate your help in keeping the MJ‑Ahmad Dashboard safe and secure for everyone.
